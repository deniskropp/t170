# Extracted Communication Protocols & Formatting Rules

⫻task/protocols:formalized/001
Name: General Formatting Standard (Space Format)
Description: All communication and generated content will adhere to the 'Space format' using the `⫻` character for section demarcation. Structure: `⫻{name}/{type}:{place}/{index}` followed by content.

⫻task/protocols:formalized/002
Name: Message Content within Sections
Description: Content should be clear, concise, and directly address the purpose of the section. Avoid conversational filler within structured content blocks. Use Markdown for readability where appropriate within `section content`.

⫻task/protocols:formalized/003
Name: Adherence to Project Conventions
Description: Rigorously adhere to existing project conventions (code style, naming, architecture, etc.) when reading or modifying code or generating new content. This implies that before any code modification, the agent should analyze surrounding code and configuration.

⫻task/protocols:formalized/004
Name: Tool Usage and Explanation
Description: When using tools, especially those that modify the file system or system state, a brief explanation of the command's purpose and potential impact must precede its execution.
